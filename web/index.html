HOW TO BECOME A FULLSTACK WEB DEVELOPER

Web applications usually have 3 main different parts; The Frontend, Backend and Database. Each separate part of the application has a different set of responsibilities for which it is meant to contribute to the entire application's functionalities. In order to be a fullstack web developer, you must have the knowledge that spans the frontend, the backend, and the database. Lets break it down so that you can get the bigger and better picture of how this can be achieved.


THE FRONTEND

This is the part of the application that the end users interact with. It main purpose is to provide the end user with some form of presentation where to interact with the application. It is basically a medium for input/output between the User and the Application. The diagram below demonstrates how the frontend(browser), the backend and the database exist in their relative positions.

There are 3 main languages to learn in order to start building this part of your application; HTML5, CSS3 and Javascript.


This blog wont dive into any particular programming language, instead I'll give you a brief explanation of what each language is responsible for. I'll include links for resources to learn more about some of the programming languages I'll be introducing in this blog.


HTML5: Every web page is made up of elements ( like text, pictures, boxes and other shapes, etc. ) which define the content of the web page. These Elements are coded into a file with a ".html" file extension and is of the html syntax. This makes it possible for the web browsers to read and create the webpages from the file. The webpage elements are represented in code with html tags. Most of such elements are made up of a start and a closing tags (like so, <startTag> content here </closeTag>). In between the start and closing tags is where to put the main content which should be displayed on the webpage. Elements have properties too, which help the browser understand what type of elements they are and how to display them on the webpage. In oder to add properties to elements, you place them in the <startTag> in the form of name="value" like so <startTag propertyname="value">. Click here for a beginners' HTML5 tutorial


CSS3: Webpages don't only have content, they also have designs. In order to make your webpage look attractive to the user, you need to write code that customize the appearances of the elements on the page. CSS3 is the language used for styling your webpage and making it look how attractive you want it to look. Example, if you want the heading to be of specific font, color, size, etc, you do that in CSS3 code, if you want a user's profile picture to be shaped as a circle instead of a rectangle, you do that in CSS3 code as well. Click here for a beginners' CSS3 tutorial


Javascript: Modern websites are interactive; that is, the web pages don't get refreshed or redirected for every user click. Users are able to stay on the same webpage and perform various activities without having to refresh the webpage. This is achieved using Javascript code. In order to add some interactivity to the webpages, you need to write javascript code which get executed inside the browser in response to various user activities. For example, when the user clicks on a drop-down menu icon, some javascript code is executed so that the actual drop-down menu appears on the screen. Javascript can be quite heavy to get your head around if you have no prior programming experience. Click here for a beginners' Javascript tutorial


As a summary of what we've covered so far, Frontend is the part of the application that the end user interacts with in order to use the application. It contains all the visual elements such as forms, buttons, menus, window panels, etc. In order to build this kind of presentation which can be displayed by the web browser, you create an html page which contains html elements written of the HTML5 language syntax, then add your design definitions using CSS3 code. Modern webpages aren't static pages with fixed contents, they allow the users to change parts of the webpages without having to refresh the pages all the time. Javascript is the language for implementing the behavioural part of the frontend.


So far we've covered the first part of the application, now lets consider looking at the backend and how its different from the frontend.


THE BACKEND

The backend of every software application is that which the end user isn't allowed to directly access its code, instead, the user only gets a feel of what exists in the backend by invoking its functions through interacting with the frontend. This part of the application is responsible for some of the most complex applications functions and processes, such as user authentication, database interaction, data processing etc. When it comes to the programming languages for building the backend, there is a vast variety of languages to learn in order to write backend code and these languages usually have steep learning curves. This blog isn't meant to teach any set of specific languages, but what I will be doing is briefly introducing some of the very popular backend languages so as to give you a feel of what the backend may comprise of. Some backend languages to condiser include Java (not the same as Javascript), Python, C#, Ruby, PHP, and even Javascript ( yes, you can use javascript on the backend too, it is called Node.js ).

As a summery of what the backend is, it is the non-visual part of your application which contains code that runs on the server side. The code in the backend is usuaslly invoked indirectly by the users through interacting with the front-end, thus some of the frontend elements or features connect to various processes and functions in the backed code. This code isn't accessible to the end user however, but through their direct actions on frontend elements, the backend code is activated. Some of the common things the backend is used for are user authentication, database read, write, update, delete, etc. operations, complex data processing, etc. Java, C#, Python, Javascript, Ruby, PHP etc. are some of the most popular backend languages.


THE DATABASE

The database is the data storage part of your application. It stores all the data that is generated by the end users and application processes in order to maintain data persistence. For example, if you need the users to create user accounts to be able to use the application, then you are going to need a place to store all this user generated data so that it can be later retrieved and used when needed. Databases are a way to permanently store data. Databases come in two main general categories. These are the Relational or SQL databases and the NoSQL databases.

Having covered all three main application concerns, the table is now set for us to look at what being a fullstack web developer means.


WHO IS A FULLSTACK WEB DEVELOPER

A fullstack web developer is someone who has knowledge of all the three main application concerns and is able to piece everything together to build the whole application. As a fullstack web developer, you are required to have the knowledge that spans all three main appliction concerns ( frontend, backend, database). You don't have to be an expert in all three, but atleast having a good understanding of one facet of the software stack and being average on all other parts is required to meet the expectations of a fullstack web developer. The most conventional way to begin your journey is to pick one particular aspect, usually the frontend, and start learning the technologies and languages involved in building that part of the application. Thus being a fullstack web developer is only the ultimate goal, therefore needless do you have to focus on learning all three areas of the application at the same time.
